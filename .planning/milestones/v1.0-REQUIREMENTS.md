## v1 Requirements

### Demand Planning

- [x] **DEM-01**: Dashboard shows cross-brand demand summary — forecasted units vs ordered units vs in-stock units per brand per month
- [x] **DEM-02**: Retailer-level breakdown showing what each retailer needs across all brands (TJX, HomeGoods, Walmart Canada, CVS, 5 Below, Ross, Burlington, Bealls, etc.)
- [x] **DEM-03**: SKU-level drill-down displaying demand, ordered, available, in-transit, and balance for every SKU
- [x] **DEM-04**: Shortage alerts auto-flag SKUs where forecasted demand exceeds ordered + available inventory
- [x] **DEM-05**: Excess alerts auto-flag SKUs where ordered/available significantly exceeds forecasted demand

### Order Tracking

- [x] **ORD-01**: Supplier PO tracking — internal POs to China showing deposit status, production status, shipping status
- [x] **ORD-02**: Retail PO tracking — incoming POs from retailers with fulfillment status
- [x] **ORD-03**: Lead time visibility showing order-by dates and expected arrival dates per SKU (30-day for Fomin/Luna/EveryMood/Roofus, 60-day for HOP)
- [x] **ORD-04**: PO lifecycle timeline — visual pipeline: ordered > in production > shipped > in transit > arrived at warehouse > allocated

### Dashboard & UX

- [x] **UX-01**: Brand selector allowing users to view all brands or filter to a single brand
- [x] **UX-02**: Role-based views — CEO overview, sales view, purchasing view, warehouse view
- [x] **UX-03**: Executive summary page — one-screen health check with top alerts, key numbers, and action items
- [x] **UX-04**: Clean, visual design usable by non-technical team members without training
- [x] **UX-05**: Navigation organized by brand and by function (demand, orders, cash flow)

### Data Integration

- [x] **DAT-01**: Excel forecast import — parse RTL FORECAST_MASTER format (Fomin, Luna, EveryMood, Roofus) and HOP product-centric format
- [x] **DAT-02**: SellerCloud API integration — pull POs, inventory levels, shipment tracking, and payment status automatically
- [x] **DAT-03**: Manual data entry — team can create/update orders and status when API data is incomplete or unavailable
- [x] **DAT-04**: Retail sales data import — upload retail sales Excel files for SKU performance tracking
- [x] **DAT-05**: Data validation on import — flag errors, show preview before committing, reject malformed data

### Foundation

- [x] **FND-01**: Database schema supporting 5 brands, ~160 SKUs, ~12 retailers, POs, inventory, forecasts, payments
- [x] **FND-02**: User authentication with role assignment (CEO, sales, purchasing, warehouse)
- [x] **FND-03**: Master data management — brands, SKUs, retailers as canonical reference data
- [x] **FND-04**: Audit log tracking who changed what data and when
